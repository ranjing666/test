# 1000 天详细学习计划

这不是一个只有标题的概览，而是一套按天执行的长期学习路线。

适合对象：
- 中文母语
- 编程零基础或基础很弱
- 英语基础弱但愿意长期积累
- 希望把学习内容持续同步到 GitHub

## 计划怎么用

每天建议 90 到 150 分钟：
- 20 分钟：英语词汇和句型
- 45 到 80 分钟：编程主线学习和敲代码
- 15 到 30 分钟：练习、复盘、写学习日志
- 10 分钟：提交代码或整理笔记

如果当天很忙，不要完全停掉，至少完成这 3 件事：
- 看当天计划
- 敲 10 分钟代码
- 在日志里写 3 句话

## 学习原则

1. 先理解最小例子，再自己改例子。
2. 报错不是失败，是学习材料。
3. 每 10 天做一个小项目或整合练习。
4. 编程和英语同时推进，但每次只抓少量重点。
5. 只要不断更，就能逐渐形成自己的 GitHub 学习档案。

## 现在立刻该做什么

1. 看 `stage1_foundation/units/unit_001.md`。
2. 只执行里面的 `Day 001`，不要急着看很远。
3. 完成后填写 `study_logs/day001.md`。
4. 学完几天后，再回头看整体节奏。
