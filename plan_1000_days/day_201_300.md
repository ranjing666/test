# Day 201-300

阶段主题：前端与网页基础

阅读方式：
- 只执行当前这一天。
- 每 10 天做一个小整合。
- 看不懂时，先跑例子，再回头读解释。

## 单元 021：Git 与 GitHub 基础（Day 201-210）
- 编程焦点：仓库概念；git status add commit；远程仓库 push pull；写清晰提交信息
- 英语词汇：git, commit, push, pull, branch, repo
- 英语句型：`I can save my work with Git.`
- 本单元产出：把前面的小项目整理并推送到 GitHub

### Day 201
- 编程：先理解 `仓库概念`，只求知道它是什么、为什么要学。
- 英语：朗读并认识这些词：git, commit, push, pull, branch, repo。
- 任务：读 3 个最小例子；手敲 1 遍；再用中文写出对“Git 与 GitHub 基础”的第一理解。
- 产出：在学习日志中整理 5 个关键词，并说明 `仓库概念` 的作用。
- 检查：能用自己的话解释 `仓库概念` 是什么。

### Day 202
- 编程：围绕 `仓库概念` 做跟练，把昨天看到的内容变成自己能敲出来的代码。
- 英语：熟悉句型：`I can save my work with Git.`，并尝试自己读 3 遍。
- 任务：重写昨天的例子，主动改 3 个小地方，例如文字、数字、顺序或变量名。
- 产出：得到一个能运行的练习版本，作为“把前面的小项目整理并推送到 GitHub”的第 1 个小部件。
- 检查：能独立写出一个包含 `仓库概念` 的最小例子。

### Day 203
- 编程：开始学习 `git status add commit`，注意它和前一个知识点的关系。
- 英语：把今天遇到的高频词写成一个小词表，并给出中文解释。
- 任务：先看例子，再自己模仿 2 次，重点观察 `git status add commit` 在代码中的位置。
- 产出：补一段学习笔记，说明 `仓库概念` 和 `git status add commit` 分别负责什么。
- 检查：能指出一段代码里哪部分属于 `git status add commit`。

### Day 204
- 编程：把 `仓库概念` 和 `git status add commit` 放在一起练，开始做小题或小改造。
- 英语：把一个英文句子拆开理解，至少认出主语、动词和关键技术词。
- 任务：完成 3 个小练习，每个练习只改一处核心逻辑，避免一次改太多。
- 产出：整理一页错题或疑问记录，写出哪里容易混淆。
- 检查：能自己找出 `仓库概念` 和 `git status add commit` 的配合方式。

### Day 205
- 编程：学习 `远程仓库 push pull`，开始接触更接近真实程序的写法。
- 英语：围绕今天的主题写 3 句很短的英文说明，不追求复杂，只求准确。
- 任务：看示例、仿写、运行，再自己解释为什么要用 `远程仓库 push pull`。
- 产出：写一段 80 到 120 字的中文总结，主题是 `远程仓库 push pull`。
- 检查：能说明 `远程仓库 push pull` 解决了什么问题。

### Day 206
- 编程：把 `远程仓库 push pull` 和前面的内容结合，完成一组更完整的练习。
- 英语：复习本单元词汇，并把其中 5 个词写进代码注释或学习记录。
- 任务：做 2 到 3 个组合练习，重点是先预测结果，再运行验证。
- 产出：形成“把前面的小项目整理并推送到 GitHub”的练习版雏形。
- 检查：能在不看答案的情况下完成一次跟练。

### Day 207
- 编程：学习 `写清晰提交信息`，为本单元小项目补上最后一块核心能力。
- 英语：听读一句英文描述，再用中文解释这句话的意思和关键词。
- 任务：先写一个最小例子，再把 `写清晰提交信息` 放进正在做的小项目中。
- 产出：列出“把前面的小项目整理并推送到 GitHub”还差哪些功能，形成一个很小的待办清单。
- 检查：能说清 `写清晰提交信息` 在项目里负责哪部分。

### Day 208
- 编程：进入本单元项目开发日，把 `仓库概念`、`git status add commit`、`远程仓库 push pull`、`写清晰提交信息` 串起来。
- 英语：把本单元句型和词汇复述一遍，并尝试口头讲解今天写的代码。
- 任务：围绕“把前面的小项目整理并推送到 GitHub”完成输入、处理、输出三部分，先求能运行，再求美观。
- 产出：得到“把前面的小项目整理并推送到 GitHub”的第 1 个完整可运行版本。
- 检查：能从头到尾演示一次程序运行流程。

### Day 209
- 编程：进入调试和完善日，重点是修 Bug、改命名、补注释、删重复。
- 英语：用简单英文写 3 到 5 句项目说明，例如程序做什么、怎么运行、输入什么。
- 任务：回放报错过程，记录 3 个今天修掉的问题，并写下修复方法。
- 产出：得到“把前面的小项目整理并推送到 GitHub”的优化版，并配一段中英结合说明。
- 检查：能说出至少 3 个自己今天修掉或避免的问题。

### Day 210
- 编程：做 `Day 201-210` 总复习，把本单元知识点串成一张小地图。
- 英语：把本单元高频词再读一遍，并复述句型，力求看见就懂大意。
- 任务：整理笔记、运行最终版、提交 GitHub、写周报式总结，并预热下一单元。
- 产出：提交“把前面的小项目整理并推送到 GitHub”最终版，外加一份阶段复盘。
- 检查：能口头复述本单元 5 到 10 个关键点。
