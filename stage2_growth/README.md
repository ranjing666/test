# 第 2 阶段：成长进阶

这一阶段对应 `Day 101-300`，重点是把“Python 基础能力”继续推到“能开始做网页、能开始连接 GitHub、能开始接触前后端入口”。

## 这一阶段现在学什么

这一阶段的实际单元范围是：

- `unit_011` 到 `unit_030`

学习顺序不是乱跳的，而是这样衔接：

1. 先把 Python 基础数据结构和常用能力补全
2. 再学文件、调试、模块、类和标准库
3. 然后接 Git 和 GitHub
4. 再进入 HTML、CSS、JavaScript、DOM
5. 最后进入 Flask 路由，作为后端入口

## 本阶段核心目标

完成这一阶段后，你应该能做到：

- 会使用列表、字典、字符串常用方法
- 会读写文件，知道怎么定位和修复常见错误
- 知道模块、类、对象和标准库的作用
- 会使用 Git 和 GitHub 保存学习过程
- 能写基础网页结构和简单样式
- 能写基础 JavaScript 交互
- 能理解最基础的 Flask 网站入口

## 单元结构

### 单元 011-020
- 列表、字典、字符串、文件、调试、模块、类、标准库
- 目标：把 Python 基础能力练到可以支持小项目

### 单元 021-030
- Git / GitHub、HTML、CSS、JavaScript、DOM、前端小项目、Flask 路由
- 目标：从命令行程序过渡到网页和简单 Web 开发

## 建议学习顺序

固定按这个顺序学，不容易乱：

1. 先看 `units/unit_011.md`
2. 再做 `workbooks/unit_011_workbook.md`
3. 然后补 `code_templates/unit_011_template.md`
4. 卡住时再看 `code_solutions/unit_011_solution.md`
5. 最后做 `quizzes/unit_011_quiz.md`
6. 写入 `study_logs/`
7. 在 `LEARNING_PROGRESS_TRACKER.md` 打勾

## 学这一阶段时最容易犯的错

- 只想快点做网页，结果 Python 基础没补稳
- 一看到 HTML/CSS/JS 就把前面的调试和文件处理忘掉
- 只做页面，不写 Git 提交和学习记录
- 一开始就追求复杂页面，忽略最小可运行版本

## 这一阶段的原则

- 先把功能做通，再慢慢变好看
- 先让页面能显示，再让页面能交互
- 先会最小的 Git 流程，再谈协作
- 先明白“输入、处理、输出”，再做更完整项目
