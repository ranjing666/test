## 本单元先做什么

1. 先看单元讲义，知道为什么学。
2. 再看模板，照着补一个最小版本。
3. 做下面的练习，不要求一步到位。
4. 最后再对照参考答案和小测。

## 本单元目标

- 知道 `README 结构` 在当前单元里负责什么。
- 知道 `Issue 和分支` 在当前单元里负责什么。
- 知道 `Pull Request 概念` 在当前单元里负责什么。
- 知道 `代码审查意识` 在当前单元里负责什么。
- 最后做出：`为一个项目写完整 README 和使用说明`

## 热身问题

1. 你能不能用一句大白话解释 `README 结构`？
2. 你知不知道 `Issue 和分支` 和前面内容的关系？
3. 你能不能说出最终产出 `为一个项目写完整 README 和使用说明` 的最小版本应该长什么样？

## 跟着做

1. 先把模板里的第一段内容手敲一遍，感受 `README 结构`。
2. 再补上第二块内容，感受 `Issue 和分支`。
3. 再用你自己的话改一个小地方，把它和 `Pull Request 概念` 接起来。

## 基础练习

1. 做一个最小版本，只要能体现 `README 结构` 和 `Issue 和分支` 就行。
2. 再补上 `Pull Request 概念`，让结果更完整。
3. 最后补上 `代码审查意识`，让它更接近 `为一个项目写完整 README 和使用说明`。

## 提升练习

1. 把命名改得更清楚。
2. 补一句英文说明，解释这个单元的产出在做什么。
3. 如果这是网页或接口，就让它多一块可见结果；如果这是算法或数据题，就多做一组输入输出。

## 英语练习

- 本单元词汇：`readme`, `issue`, `branch`, `review`, `document`, `team`
- 本单元句子：`Good documentation helps teamwork.`
- 先大声读 3 遍，再用中文讲出大意。
- 任选 3 个词，写 3 个很短的技术句子。

## 小项目推进建议

- 第一步：先完成 `为一个项目写完整 README 和使用说明` 的最小可运行版。
- 第二步：再让 `Pull Request 概念` 和 `代码审查意识` 进入成果里。
- 第三步：最后再整理命名、注释、说明文字。
