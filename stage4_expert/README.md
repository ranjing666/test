# 第 4 阶段：专家深化

这一阶段对应 `Day 601-850`，重点是把能力推进到现代前端、现代后端、部署和工程质量层面。

## 这一阶段现在学什么

这一阶段的实际单元范围是：

- `unit_061` 到 `unit_085`

整体路线是：

1. Node、npm、React、TypeScript、前端项目
2. FastAPI、Pydantic、SQLAlchemy、JWT、异步、Docker、Linux、CI/CD、云部署
3. 重构、设计模式、测试进阶、性能优化、稳定性和队列

## 本阶段核心目标

完成这一阶段后，你应该能做到：

- 会搭建现代前端开发环境
- 会用 React 和 TypeScript 做更完整的前端项目
- 会用 FastAPI 搭建现代 Python 接口
- 会理解容器、Linux、CI/CD、云部署的最小流程
- 会从“能写代码”进一步走向“能写得更稳、更清楚、更可维护”

## 单元结构

### 单元 061-070
- 前端工具链、React、TypeScript、前端项目
- 目标：形成现代前端开发能力

### 单元 071-080
- FastAPI、鉴权、异步、Docker、Linux、CI/CD、云部署、全栈集成
- 目标：形成现代后端与上线能力

### 单元 081-085
- 重构、设计模式、测试、性能、稳定性
- 目标：形成工程质量意识

## 建议学习顺序

1. 先看 `daily_guides/day_601.md`
2. 再看 `units/unit_061.md`
3. 再做 `workbooks/unit_061_workbook.md`
4. 然后补 `code_templates/unit_061_template.md`
5. 卡住时再看 `code_solutions/unit_061_solution.md`
6. 最后做 `quizzes/unit_061_quiz.md`
7. 继续写 `study_logs/`
8. 更新 `LEARNING_PROGRESS_TRACKER.md`

## 这一阶段最容易出现的问题

- 一看到 React / FastAPI / Docker 就急着想一步到位
- 对工具链和部署环境不熟时，容易把“环境问题”误以为“不会编程”
- 写完功能后，不愿意回头做重构、测试和说明
- 只想看结果，不愿意理解为什么要这样组织工程

## 这一阶段的原则

- 先把项目跑起来，再优化结构
- 先会一个完整最小流程，再补更多配置
- 先理解工程为什么复杂，再慢慢学会驾驭复杂
- 每完成一个单元，至少留下一个可以回看的成果
