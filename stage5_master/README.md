# 第 5 阶段：大师输出

这一阶段对应 `Day 851-1000`，重点是把你前面学到的技术收束成“能展示、能表达、能交付、能长期发展”的综合能力。

## 这一阶段现在学什么

这一阶段的实际单元范围是：

- `unit_086` 到 `unit_100`

整体路线是：

1. 系统设计、日志、监控、安全、开源贡献
2. 专项项目、作品集、技术写作、技术表达、简历、面试
3. 毕业大项目规划、开发、发布、复盘

## 本阶段核心目标

完成这一阶段后，你应该能做到：

- 能把一个技术方案讲清楚
- 能整理作品集、简历和项目表达材料
- 能更系统地理解监控、安全、设计和交付
- 能完成一个毕业级大项目
- 能把前面 1000 天的积累收束成长期可用的能力资产

## 单元结构

### 单元 086-090
- 系统设计、监控、安全、开源、专项项目
- 目标：开始站在更高层看系统和长期成长

### 单元 091-095
- 作品集、技术写作、演示、简历、面试
- 目标：把技术能力变成可展示、可表达、可求职的成果

### 单元 096-100
- 毕业项目规划、开发、发布、复盘
- 目标：完成你的 1000 天收官作品

## 建议学习顺序

1. 先看 `daily_guides/day_851.md`
2. 再看 `units/unit_086.md`
3. 再做 `workbooks/unit_086_workbook.md`
4. 然后补 `code_templates/unit_086_template.md`
5. 卡住时再看 `code_solutions/unit_086_solution.md`
6. 最后做 `quizzes/unit_086_quiz.md`
7. 持续写 `study_logs/`
8. 更新 `LEARNING_PROGRESS_TRACKER.md`

## 这一阶段最重要的提醒

- 不要把“输出”和“表达”当成附属品，它们本身就是能力
- 不要只想把项目做完，还要能说明白为什么这么做
- 简历、作品集、演示和大项目，是你 1000 天学习成果的浓缩
- 最后这段不只是冲刺，更是系统整理

## 这一阶段的原则

- 先把主线成果做完整，再补漂亮细节
- 先说清问题和结果，再谈炫技
- 先形成自己的方法，再想着帮助别人
- 每完成一个单元，都留下能展示的材料
