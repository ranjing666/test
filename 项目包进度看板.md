# 项目包进度追踪表

这个文件专门记录你在各个项目包里的推进状态。

不要等“全部做完”才记录。项目一旦开始，就应该持续更新。

## 状态说明

- `not_started`：还没开始
- `running`：已经能跑起来
- `testing`：正在按测试清单检查
- `polishing`：正在改结构、说明、样式或细节
- `done`：当前版本已达标

## 每次更新时写什么

至少写 3 件事：

1. 现在在哪个状态
2. 当前版本号
3. 这次解决了什么问题

## 建议版本写法

- `v0.1`：刚跑通
- `v0.2`：完成主要功能
- `v0.3`：补完测试或说明
- `v1.0`：达到当前单元标准

## 什么时候可以把状态改成 done

至少满足：

- 跑通
- 自测通过
- 读得懂自己的结构
- 知道下一步还能怎么改
